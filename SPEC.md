## 1. Pipeline Overview

### Stages at a Glance

| Stage | Name | Trigger | Output | Human Gate? |
|-------|------|---------|--------|-------------|
| 0 | Weekly Research | Cron (Mon 08:00 GST) | 3–5 story concepts via Telegram | No |
| 1 | Concept Selection | Darl picks via Telegram | Approved concept → pipeline_runs row | ✅ Darl |
| 2 | Script Generation | Stage 1 complete | JSON script via Telegram for review | ✅ Darl |
| 3 | Character Prep | Stage 2 approved | Characters confirmed from library | No (auto, unless missing) |
| 4 | Scene Illustration | Stage 3 complete | 1 image per scene in Supabase Storage | No (auto) |
| 5 | Animation | Stage 4 complete | 1 video clip per scene | No (auto) |
| 6 | Tamil Voice (TTS) | Stage 5 complete | Audio files per scene (2 takes) | No (auto) |
| 7 | Assembly | Stage 6 complete | Final MP4 via ffmpeg | No (auto) |
| 8 | Human Review | Stage 7 complete | Telegram notification for Darl review | ✅ Darl |
| 9 | Feedback + Publish | Stage 8 approved | Video published to YouTube | ✅ Final confirmation |

### Pipeline State Machine

```
[PENDING] → [RUNNING] → [AWAITING_REVIEW] → [APPROVED] → [RUNNING] → [COMPLETED]
                ↓                                  ↓
            [FAILED]                           [REJECTED] → [RUNNING] (retry with feedback)
```

Each stage is a discrete unit tracked in `video_pipeline_runs`. The pipeline waits for Telegram approval signals (via Heimdall bot) before advancing.

### 2.1 `character_library`

Stores all character definitions. The special `NARRATOR` entry has `NULL` image fields.

```sql
CREATE TABLE character_library (
  id                  UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  name                TEXT          NOT NULL UNIQUE,         -- e.g. "Velu", "NARRATOR"
  description         TEXT          NOT NULL,                -- plain-text character description for Claude context
  reference_image_url TEXT,                                  -- NULL for NARRATOR
  image_prompt        TEXT,                                  -- NULL for NARRATOR; base prompt for image gen
  voice_id            TEXT          NOT NULL,                -- ElevenLabs voice_id
  approved            BOOLEAN       NOT NULL DEFAULT FALSE,
  feedback            TEXT[]        NOT NULL DEFAULT '{}',   -- array of human feedback strings
  version             INTEGER       NOT NULL DEFAULT 1,      -- increments on each approved update
  created_at          TIMESTAMPTZ   NOT NULL DEFAULT NOW(),
  updated_at          TIMESTAMPTZ   NOT NULL DEFAULT NOW()
);

-- Index for quick approved-only lookups
CREATE INDEX idx_character_library_approved ON character_library (approved);

-- Auto-update updated_at
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER character_library_updated_at
  BEFORE UPDATE ON character_library
  FOR EACH ROW EXECUTE FUNCTION update_updated_at_column();
```

**NARRATOR seed row:**
```sql
INSERT INTO character_library (name, description, reference_image_url, image_prompt, voice_id, approved)
VALUES (

  'The story narrator. Warm, gentle, storytelling voice. No visual representation.',
  NULL,
  NULL,
  'ELEVENLABS_NARRATOR_VOICE_ID',  -- replace at seed time

```

**Field notes:**
- `image_prompt`: the *base* visual prompt used for scene illustration. The pipeline appends scene-specific descriptors at generation time. Feedback engine may update this column automatically after analysis.
- `feedback`: appended (not overwritten) via `array_append()`. Preserved across versions for audit.
- `version`: incremented when `image_prompt` or `voice_id` changes after human approval.

### 2.2 `video_pipeline_runs`

One row per stage per video. The orchestrator writes here before and after each stage.

```sql
CREATE TABLE video_pipeline_runs (
  id            UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  task_id       UUID          NOT NULL,                   -- the parent video task (groups all stage rows)
  stage         INTEGER       NOT NULL CHECK (stage BETWEEN 0 AND 9),
  status        TEXT          NOT NULL DEFAULT 'pending'
                              CHECK (status IN ('pending','running','completed','failed','awaiting_review','approved','rejected')),
  cost_usd      NUMERIC(10,6) NOT NULL DEFAULT 0,         -- actual cost incurred in this stage
  error         TEXT,                                     -- last error message if status = 'failed'
  started_at    TIMESTAMPTZ,
  completed_at  TIMESTAMPTZ
);

CREATE INDEX idx_pipeline_runs_task_id ON video_pipeline_runs (task_id);
CREATE INDEX idx_pipeline_runs_status  ON video_pipeline_runs (status);
```

**Field notes:**
- `task_id`: shared UUID that groups all stages for a single video. Generated at Stage 0 concept approval.
- `cost_usd`: stage-level cost. Sum all rows for a `task_id` to get total video cost.
- A separate view is useful for monitoring:

```sql
CREATE VIEW video_cost_summary AS
SELECT
  task_id,
  SUM(cost_usd) AS total_cost_usd,
  MAX(stage)    AS current_stage,
  COUNT(*)      AS stage_count
FROM video_pipeline_runs
GROUP BY task_id;
```

## 3. Script Format

Claude (claude-sonnet-4-6 or higher) must produce **exactly** this JSON structure. Any deviation is a pipeline error (Stage 2 fails).

The system prompt instructs Claude to return **only** valid JSON with no markdown wrapping.

### 3.1 Full JSON Schema

```jsonc
{
  // ── METADATA ──────────────────────────────────────────────────────────────
  "metadata": {
    "title":                   "கரடி மற்றும் தேனீ",         // Tamil story title
    "episode":                 7,                              // integer episode number
    "target_duration_seconds": 300,                            // total video target (e.g. 300 = 5 min)
    "characters": [                                            // all characters appearing in this episode
      "NARRATOR",
      "Velu",
      "Meena"
    ]
  },

  // ── YOUTUBE SEO ───────────────────────────────────────────────────────────
  "youtube_seo": {
    "title":       "கரடி மற்றும் தேனீ | Bear and Bee | Tamil Kids Story | Tiny Tamil Tales",
    "description": "...(200-400 chars, Tamil first, English subtitle, channel CTA)...",
    "tags": [
      "tamil kids story", "tamil animated story", "kids story in tamil",
      "tiny tamil tales", "சிறுவர் கதை", "தமிழ் கதை"
      // 10-15 tags, mix Tamil + English
    ]
  },

  // ── SCENES ────────────────────────────────────────────────────────────────
  "scenes": [
    {
      "scene_number":             1,
      "visual_description":       "A lush green forest at golden hour. A small brown bear cub named Velu sits near a hollow tree, ears perked, looking curious.",
      "motion_type":              "landscape",               // one of: dialogue | action | landscape | emotional
      "estimated_duration_seconds": 30,

      "lines": [
        {
          "speaker": "NARRATOR",
          "text":    "ஒரு காட்டில் வேலு என்ற சிறிய கரடி குட்டி வாழ்ந்தது.",
          "emotion": "warm"                                   // see emotion vocabulary below
        }
      ]
    },
    {
      "scene_number":             2,
      "visual_description":       "Close-up of Velu's face looking upward at a buzzing beehive. His eyes are wide and shiny. A single bee hovers near his nose.",
      "motion_type":              "dialogue",
      "estimated_duration_seconds": 25,

      "lines": [
        {
          "speaker": "Velu",
          "text":    "ஆஹா! அந்த தேனீ என்னிடம் பேசுகிறதா?",
          "emotion": "surprised"
        },
        {
          "speaker": "NARRATOR",
          "text":    "வேலு ஆச்சரியப்பட்டான்.",
          "emotion": "warm"
        }
      ]
    }
    // ... repeat for all scenes
  ]
}
```

### 3.2 Motion Type Vocabulary

| `motion_type` | When to use | Kling behaviour |
|---------------|-------------|-----------------|
| `dialogue`    | Characters talking, close-up / medium shot | Subtle lip/body movement, stable camera |
| `action`      | Characters running, jumping, interacting physically | Dynamic motion, camera follows |
| `landscape`   | Establishing shots, scenery, no main character focus | Slow pan or parallax drift |
| `emotional`   | Reaction shots, sad/happy moments, close-up emotion | Gentle breathing/expression movement |

### 3.3 Emotion Vocabulary (lines[].emotion)

Allowed values (used for ElevenLabs SSML `<amazon:emotion>` approximation):
`warm` | `happy` | `sad` | `surprised` | `scared` | `excited` | `calm` | `curious` | `angry` | `proud`

#### Image Generation API Call

```javascript
// lib/image-gen.mjs
import { GoogleGenerativeAI } from "@google/generative-ai";

const genAI = new GoogleGenerativeAI(process.env.GOOGLE_AI_API_KEY);

export async function generateSceneImage({ prompt, characterName }) {
  // Fetch model from pipeline_settings
  const model = await getSetting("image_model"); // e.g. "imagen-3.0-fast-generate-001"
  
  const response = await fetch(
    `https://generativelanguage.googleapis.com/v1beta/models/${model}:predict?key=${process.env.GOOGLE_AI_API_KEY}`,
    {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        instances: [{ prompt }],
        parameters: {
          sampleCount: 1,
          aspectRatio: "16:9",           // YouTube landscape
          safetyFilterLevel: "block_few", // kids content — keep moderate
          personGeneration: "dont_allow"  // no real people
        }
      })
    }
  );

  const data = await response.json();
  if (!response.ok) throw new Error(data.error?.message || "Image gen failed");

  // Returns base64 PNG
  const base64 = data.predictions[0].bytesBase64Encoded;
  return Buffer.from(base64, "base64");
}
```

```javascript
// lib/motion-params.mjs
export function getKlingParams(motionType, settingsOverrides = {}) {
  const defaults = {
    dialogue:  { duration: 5, cfg_scale: 0.4, mode: "std",
                 prompt_suffix: "character talking gently, subtle body movement, stable frame" },
    action:    { duration: 5, cfg_scale: 0.6, mode: "std",
                 prompt_suffix: "dynamic movement, camera follows character, lively motion" },
    landscape: { duration: 5, cfg_scale: 0.3, mode: "std",
                 prompt_suffix: "slow parallax pan, peaceful drift, establishing mood" },
    emotional: { duration: 5, cfg_scale: 0.5, mode: "std",
                 prompt_suffix: "gentle breathing, subtle facial expression, close emotional moment" }
  };

  const base = defaults[motionType] ?? defaults.dialogue;
  // settingsOverrides come from pipeline_settings.kling_motion_type_defaults
  return { ...base, ...settingsOverrides[motionType] };
}
```

**Polling for completion:**
```javascript
// Kling jobs are async. Poll until done.
// GET https://api.kie.ai/v1/videos/image2video/{task_id}
// Response: { status: "processing" | "completed" | "failed", video_url: "..." }

export async function pollKlingJob(taskId, maxWaitMs = 180000) {
  const start = Date.now();
  while (Date.now() - start < maxWaitMs) {
    const res = await fetch(`https://api.kie.ai/v1/videos/image2video/${taskId}`, {
      headers: { Authorization: `Bearer ${process.env.KIEAI_API_KEY}` }
    });
    const data = await res.json();
    if (data.status === "completed") return data.video_url;
    if (data.status === "failed") throw new Error(`Kling job failed: ${data.error}`);
    await sleep(5000); // poll every 5s
  }
  throw new Error(`Kling job timed out after ${maxWaitMs}ms`);
}
```

- `kling-v1-5` std mode, 5s clip: ~$0.14/clip

### 4.3 ElevenLabs — Multi-Voice Tamil TTS

**API base:** `https://api.elevenlabs.io/v1`  
**Auth:** `xi-api-key: {ELEVENLABS_API_KEY}`

#### Voice Assignment

Each character in `character_library` has a `voice_id`. The pipeline segments the script by speaker and routes each segment to the correct voice.

#### TTS Request Per Line

```javascript
// lib/tts.mjs
export async function generateSpeech({ text, voiceId, emotion, settings }) {
  const ssml = buildSSML(text, emotion, settings);

  const response = await fetch(
    `https://api.elevenlabs.io/v1/text-to-speech/${voiceId}/stream`,
    {
      method: "POST",
      headers: {
        "xi-api-key": process.env.ELEVENLABS_API_KEY,
        "Content-Type": "application/json"
      },
      body: JSON.stringify({
        text: ssml,
        model_id: "eleven_multilingual_v2",   // supports Tamil
        voice_settings: {
          stability:        settings.tts_stability,        // from pipeline_settings
          similarity_boost: settings.tts_similarity_boost,
          style:            settings.tts_style,
          use_speaker_boost: true
        },
        output_format: "mp3_44100_128"
      })
    }
  );

  if (!response.ok) {
    const err = await response.json();
    throw new Error(`ElevenLabs error: ${JSON.stringify(err)}`);
  }

  return Buffer.from(await response.arrayBuffer());
}
```

#### SSML Builder for Kids Pacing

```javascript
function buildSSML(text, emotion, settings) {
  const rate = emotion === "excited" ? "medium"
             : emotion === "calm"    ? "x-slow"
             : settings.ssml_defaults?.narrator_rate ?? "slow";

  const pauseMs = settings.ssml_defaults?.pause_between_lines_ms ?? 600;

  // ElevenLabs supports a subset of SSML
  return `<speak>
  <prosody rate="${rate}">
    ${escapeXml(text)}
  </prosody>
  <break time="${pauseMs}ms"/>
</speak>`;
}
```

**Notes:**
- ElevenLabs eleven_multilingual_v2 supports Tamil. Test voice_id selection carefully — not all voices handle Tamil equally.
- `use_speaker_boost: true` improves clarity for kids listening.
- Kids pacing rule: default rate = `slow` for NARRATOR, `medium` for characters.

#### 2 Takes + Auto-Select

```javascript
// lib/tts-takes.mjs
export async function generateTwoTakesAndPick({ text, voiceId, emotion, settings }) {
  const [take1, take2] = await Promise.all([
    generateSpeech({ text, voiceId, emotion, settings }),
    generateSpeech({ text, voiceId, emotion, settings })  // same params, natural variation
  ]);

  // Auto-select: pick the shorter take (ElevenLabs sometimes adds trailing silence)
  // A more sophisticated selector would analyse energy/silence ratios via ffprobe
  const dur1 = await getAudioDurationMs(take1);
  const dur2 = await getAudioDurationMs(take2);

  return dur1 <= dur2 ? take1 : take2;
}

async function getAudioDurationMs(buffer) {
  // Write to tmp, ffprobe, parse
  const tmpPath = `/tmp/take-${Date.now()}.mp3`;
  await fs.writeFile(tmpPath, buffer);
  const result = execSync(
    `ffprobe -v error -show_entries format=duration -of csv=p=0 "${tmpPath}"`
  ).toString().trim();
  await fs.unlink(tmpPath);
  return parseFloat(result) * 1000;
}
```

**Cost tracking:**
- ElevenLabs charges per character billed (11 Multilingual v2: ~$0.30 per 1000 chars)
- 2 takes = 2× character count billed
- Track: `totalCharsForVideo * 2 * 0.0003` in Stage 6 cost

#### Upload as Unlisted (Stage 8 review)

```javascript
// lib/youtube.mjs
import { google } from "googleapis";

export async function uploadVideoUnlisted({ videoPath, script, taskId }) {
```

### 5.3 ElevenLabs SSML Defaults Updates

After 5 videos, if audio denial rate for a specific emotion or speaker type is >30%:
1. Analyze comments for patterns (rate too fast, pause too short, etc.).
2. Adjust `ssml_defaults` in `pipeline_settings` accordingly.
3. Log change to `pipeline_feedback` with `stage = 6, decision = 'approved', comment = "auto-adjusted by feedback engine"`.

```javascript
async function updateSSMLDefaults(audioFeedbackRows) {
  const currentDefaults = await getSetting("ssml_defaults");

  // Simple heuristic: if majority of comments contain "too fast"
  const tooFastCount = audioFeedbackRows.filter(r =>
    r.comment?.toLowerCase().includes("too fast")
  ).length;

  if (tooFastCount / audioFeedbackRows.length > 0.4) {
    currentDefaults.narrator_rate = "x-slow";
    currentDefaults.pause_between_lines_ms += 100;
  }

  await setSetting("ssml_defaults", currentDefaults);
}
```

### 5.4 Kling Motion Type Defaults Updates

After 5 videos, if a specific `motion_type` has >40% denial rate:
1. Adjust `cfg_scale` down (less motion) or `duration` up.
2. Write updated values to `pipeline_settings.kling_motion_type_defaults`.

```javascript
async function updateKlingDefaults(animationFeedbackRows) {
  const current = await getSetting("kling_motion_type_defaults");

  const byType = groupBy(animationFeedbackRows, r => r.comment_tag); // requires tagging in NEXUS

      current[motionType].cfg_scale = Math.max(0.1, current[motionType].cfg_scale - 0.1);
    }
  }

  await setSetting("kling_motion_type_defaults", current);
}
```

### 6.2 Cost Tracking Implementation

```javascript
// lib/cost-tracker.mjs

export class CostTracker {
  constructor(taskId) {
    this.taskId = taskId;
    this.stageCosts = {};
  }

  addCost(stage, amountUsd) {
    this.stageCosts[stage] = (this.stageCosts[stage] ?? 0) + amountUsd;
  }

  async flush(stage) {
    await supabase
      .from("video_pipeline_runs")
      .update({ cost_usd: this.stageCosts[stage] ?? 0 })
      .eq("task_id", this.taskId)
      .eq("stage", stage);
  }

  async totalSpent() {
    const { data } = await supabase
      .from("video_pipeline_runs")
      .select("cost_usd")
      .eq("task_id", this.taskId);
    return data.reduce((sum, row) => sum + parseFloat(row.cost_usd), 0);
  }

  async checkBudget() {
    const spent = await this.totalSpent();
    const target = parseFloat(await getSetting("budget_target_usd"));
    const cap    = parseFloat(await getSetting("budget_hard_cap_usd"));

    if (spent >= cap) {
      throw new BudgetCapExceededError(
        `Hard cap $${cap} exceeded. Spent: $${spent.toFixed(4)}. Pipeline halted.`
      );
    }

    if (spent >= target) {
      await alertDarl(
        `⚠️ Budget target $${target} reached (spent: $${spent.toFixed(4)}). Pipeline continuing to hard cap of $${cap}.`
      );
    }

    return { spent, target, cap, overBudget: spent >= target };
  }
}

export class BudgetCapExceededError extends Error {
  constructor(msg) { super(msg); this.name = "BudgetCapExceededError"; }
}
```

## 8. Error Handling & Retry Logic

### 8.1 Retry Policy Per Stage

| Stage | Max Retries | Retry Delay | Escalate to Darl When |
|-------|------------|-------------|----------------------|
| 0 | 3 | 10s exp backoff | 3 failures or YouTube API quota |
| 1 | N/A (human) | — | — |
| 2 | 3 | 5s | Claude API down or invalid JSON after 3 retries |
| 3 | 3 | 5s | Character not in library (cannot proceed without Darl) |
| 4 | 3 | 15s | Rate limit or API error after 3 retries |
| 5 | 3 | 30s | Kling job timeout (>3 min) after 3 retries |
| 6 | 3 | 10s | ElevenLabs quota exceeded |
| 7 | 2 | 5s | ffmpeg failure (corrupt input) |
| 8 | N/A (human) | — | — |
| 9 | 2 | 10s | YouTube upload failure |

### 8.2 Retry Implementation

```javascript
// lib/retry.mjs
export async function withRetry(fn, { maxRetries = 3, baseDelayMs = 5000, stage, taskId } = {}) {
  let lastError;
  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      return await fn();
    } catch (err) {
      lastError = err;
      if (err instanceof BudgetCapExceededError) throw err; // never retry budget errors

      console.error(`Stage ${stage} attempt ${attempt}/${maxRetries} failed:`, err.message);

      if (attempt < maxRetries) {
        const delay = baseDelayMs * Math.pow(2, attempt - 1); // exponential backoff
        await sleep(delay);
      }
    }
  }

  // All retries exhausted
  await supabase
    .from("video_pipeline_runs")
    .update({ status: "failed", error: lastError.message })
    .eq("task_id", taskId)
    .eq("stage", stage);

  await escalateToDarl(stage, taskId, lastError);
  throw lastError;
}
```

### 8.3 Partial Scene Failures

If 1 scene out of 12 fails at Stage 4 or 5:

1. Mark that scene as `failed` in `scene_assets` (add a `status` column).
2. **Do not abort the whole video.** Continue with remaining scenes.
3. After all scenes are processed, if **≤2 scenes** failed → create a NEXUS card: "Manual asset needed for scene {n}. Upload replacement, then resume pipeline."
4. If **>2 scenes** failed → escalate entire Stage to Darl, halt pipeline.

```javascript
// stages/stage-04-illustrate.mjs

  if (failed.length > 2) {
    throw new Error(`Too many scene failures (${failed.length}). Manual intervention required.`);
  }

  if (failed.length > 0) {

        taskId
      });

    // Wait for human upload before proceeding (pipeline pauses)

  }
}
```

## Appendix A: ffmpeg Assembly Command

```bash
# Stage 7: Assemble final video
# Input: scene_{nn}_combined.mp4 files (image+anim+audio merged per scene)
# Output: final.mp4

# Step 1: Merge image + audio per scene
ffmpeg -loop 1 -i scene_01_image.png -i scene_01_audio.mp3 \
  -c:v libx264 -tune stillimage -c:a aac -b:a 192k \
  -shortest -vf "scale=1920:1080:force_original_aspect_ratio=decrease,pad=1920:1080:(ow-iw)/2:(oh-ih)/2" \
  scene_01_combined.mp4

# Step 2: If animation exists, overlay it instead of still image
# (Kling output is used directly; audio is muxed in)
ffmpeg -i scene_01_anim.mp4 -i scene_01_audio.mp3 \
  -c:v copy -c:a aac -b:a 192k -shortest \
  scene_01_combined.mp4

# Step 3: Concatenate all scenes
# Create concat list:
echo "file 'scene_01_combined.mp4'" > concat_list.txt
echo "file 'scene_02_combined.mp4'" >> concat_list.txt
# ... etc

ffmpeg -f concat -safe 0 -i concat_list.txt \
  -i music.mp3 \
  -filter_complex "[1:a]volume=0.15[bg];[0:a][bg]amix=inputs=2:duration=first[aout]" \
  -map 0:v -map "[aout]" \
  -c:v libx264 -preset medium -crf 23 \
  -c:a aac -b:a 192k \
  final.mp4
```
